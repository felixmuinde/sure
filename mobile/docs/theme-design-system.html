2. Surface Backgrounds

All screens and card surfaces use absolute white in light mode and absolute black in dark mode. Material's colorScheme.surface is intentionally not used — it produces a tinted off-white/off-black that conflicts with the brand aesthetic.

ComponentLightDarkFlutter value
Scaffold background (all screens)WhiteBlackColors.white / Colors.black
Bottom navigation barWhiteBlackColors.white / Colors.black
Card / tile surfacesWhiteBlackColors.white / Colors.black
Chat input barWhiteBlackMatches scaffold, no shadow

Pattern

Use the following expression everywhere a surface color is needed:

theme.brightness == Brightness.light ? Colors.white : Colors.black

3. Shadow System

Shadows are hard diagonal (blurRadius: 0, offset: 4, 4). No soft/blurred drop shadows. Shadow color adapts to theme to maintain contrast and brand feel.

ElementLight shadow colorDark shadow colorAlpha
ISA Financing card _kDarkGreen _kGreen 0.18
ISA Instalments card _kDarkGreen _kGreen 0.18
Ask me anything tile _kPurple _kPurple 0.25
Suggested questions (chat) _kDarkGreen _kGreen 0.18
Greeting card No shadow —

Flutter snippet

BoxShadow(
  color: (theme.brightness == Brightness.light
      ? _kDarkGreen : _kGreen).withValues(alpha: 0.18),
  blurRadius: 0,
  offset: const Offset(4, 4),
)

5. Rules

DoDon't
Use Colors.white / Colors.black for surfaces Use colorScheme.surface or colorScheme.background for card/scaffold backgrounds
Use hard diagonal shadows (blurRadius: 0, offset: Offset(4,4)) Use soft/blurred shadows (blurRadius > 0) on branded tiles
Switch shadow color with theme.brightness (dark → green, light → dark green) Hardcode a single shadow color regardless of theme
Use _kPurple for AI/chat accent (Ask me anything, chat icon) Use purple on ISA financial data elements
Use _kGreen / _kDarkGreen for ISA / financial data elements Mix purple and green shadow on the same card
